# Quickstart

Get Wizard Duel running on your system and jump into your first battle.

## Prerequisites

- All dependencies installed (see [Installation](/installation))
- C++17 compatible compiler
- Basic familiarity with terminal commands

## Build and run

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/korrykatti/game.git
    cd game
    ```
  </Step>

  <Step title="Build the game">
    Compile the game using the Makefile:
    
    ```bash
    make
    ```
    
    This creates the executable at `build/game`.
    
    <Note>The build process uses g++ with C++17 standard and links Raylib, ENet, and system libraries.</Note>
  </Step>

  <Step title="Run the game">
    Launch the game:
    
    ```bash
    make run
    ```
    
    Or run the executable directly:
    
    ```bash
    ./build/game
    ```
  </Step>
</Steps>

### Try singleplayer first

1. Click **Singleplayer** to enter practice mode
2. Use **WASD** to move your wizard
3. Press **1** and left-click to cast a red spell
4. Try zooming with **Z** and **X** keys
5. Right-click to convert mana into health

<Tip>Start with singleplayer to familiarize yourself with the controls before jumping into multiplayer battles.</Tip>

## Network configuration

By default, the game uses these network settings:

```cpp
const char* SERVER_IP = "127.0.0.1";
const int SERVER_PORT = 7777;
```

<Note>These values are defined in `src/main.cpp:11-12`. For LAN play, modify `SERVER_IP` to the host's local IP address.</Note>

## Troubleshooting

<Accordion title="ENet initialization failed">
  If you see "An error occurred while initializing ENet", ensure:
  - ENet is properly installed
  - Your system has network capabilities
  - No firewall is blocking the application
</Accordion>

<Accordion title="Failed to create server host">
  Port 7777 might already be in use. Check for:
  - Another instance of the game running
  - Other applications using port 7777
  - Firewall blocking the port
</Accordion>

<Accordion title="Black screen or graphics issues">
  Ensure Raylib and OpenGL are properly installed:
  - Update graphics drivers
  - Verify OpenGL support
  - Check that assets directory exists
</Accordion>

## Makefile commands

| Command | Description |
|---------|-------------|
| `make` or `make all` | Compile the game |
| `make run` | Compile and run the game |
| `make clean` | Remove build artifacts |
